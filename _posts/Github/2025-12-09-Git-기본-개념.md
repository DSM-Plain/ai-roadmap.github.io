## 개요

Git은 분산 버전 관리 시스템이다. 이 문서에서는 Git의 핵심 개념들을 설명할 것이다.

## Git의 3가지 상태

### Working Directory (작업 디렉토리)
- 실제 파일들이 있는 곳
- 코드를 수정하고 작업하는 공간
- Git이 추적하지만 아직 커밋되지 않은 파일들

### Staging Area (스테이징 영역)
- 커밋할 파일들을 준비하는 중간 단계
- `git add` 명령으로 파일을 추가
- 원하는 변경사항만 선택적으로 커밋 가능

### Repository (저장소)
- Git이 프로젝트의 메타데이터와 객체 데이터베이스를 저장하는 곳
- 로컬 저장소: 내 컴퓨터에 있는 저장소
- 원격 저장소: Github 같은 서버에 있는 저장소

## 핵심 용어

### Commit (커밋)
- 변경사항의 스냅샷
- 각 커밋은 고유한 해시값을 가짐
- 커밋 메시지로 변경 이유를 기록

### Branch (브랜치)
- 독립적인 작업 공간
- 기본 브랜치는 보통 `main` 또는 `master`
- 새로운 기능 개발이나 버그 수정용으로 생성

### Merge (병합)
- 서로 다른 브랜치의 변경사항을 합치는 작업
- Fast-forward merge: 단순히 포인터 이동
- 3-way merge: 새로운 커밋 생성

### Remote (원격 저장소)
- 인터넷이나 네트워크에 있는 저장소
- `origin`: 기본 원격 저장소 이름
- `push`: 로컬 변경사항을 원격에 업로드
- `pull`: 원격 변경사항을 로컬로 다운로드

## Git의 작업 흐름

```
1. 파일 수정 (Working Directory)
   ↓
2. 변경사항 스테이징 (git add)
   ↓
3. 커밋 생성 (git commit)
   ↓
4. 원격 저장소에 푸시 (git push)
```

## .git 디렉토리

- Git 저장소의 핵심
- 프로젝트 루트에 숨김 폴더로 존재
- 모든 버전 정보와 메타데이터 저장
- 이 폴더를 삭제하면 Git 이력이 모두 사라짐

## .gitignore

- Git이 추적하지 않을 파일들을 지정
- 빌드 결과물, 로그, 환경 설정 파일 등
- 패턴 매칭으로 파일/폴더 제외

```
# 예시
node_modules/
*.log
.env
dist/
```

## 결론

Git의 기본 개념을 이해하면 버전 관리를 효과적으로 할 수 있다. 작업 디렉토리, 스테이징 영역, 저장소의 관계를 파악하는 것이 핵심이다.
