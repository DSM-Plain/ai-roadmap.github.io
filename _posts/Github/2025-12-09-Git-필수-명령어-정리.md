## 개요

Git을 사용하기 위한 필수 명령어들을 정리했다. 실무에서 자주 사용하는 명령어 위주로 구성했다.

## 초기 설정

### Git 설치 확인
```bash
git --version
```

### 사용자 정보 설정
```bash
# 전역 설정
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 설정 확인
git config --list
```

### 에디터 설정
```bash
git config --global core.editor "code --wait"  # VSCode
git config --global core.editor "vim"          # Vim
```

## 저장소 생성/복제

### 새 저장소 초기화
```bash
git init
```

### 원격 저장소 복제
```bash
git clone <저장소-URL>

# 특정 브랜치만 복제
git clone -b <브랜치명> <저장소-URL>
```

## 기본 작업 흐름

### 상태 확인
```bash
# 현재 상태 확인
git status

# 간략하게 보기
git status -s
```

### 파일 추가 (Staging)
```bash
# 특정 파일 추가
git add <파일명>

# 모든 변경사항 추가
git add .

# 특정 디렉토리 추가
git add <디렉토리명>/

# 대화형 추가
git add -p
```

### 커밋
```bash
# 커밋
git commit -m "커밋 메시지"

# 스테이징과 커밋 동시에
git commit -am "커밋 메시지"

# 마지막 커밋 수정
git commit --amend

# 마지막 커밋 메시지만 수정
git commit --amend -m "새 메시지"
```

## 브랜치 관리

### 브랜치 확인
```bash
# 로컬 브랜치 목록
git branch

# 원격 브랜치 포함
git branch -a

# 각 브랜치의 마지막 커밋 표시
git branch -v
```

### 브랜치 생성/전환
```bash
# 브랜치 생성
git branch <브랜치명>

# 브랜치 전환
git checkout <브랜치명>

# 브랜치 생성 + 전환
git checkout -b <브랜치명>

# 새로운 방식 (Git 2.23+)
git switch <브랜치명>
git switch -c <브랜치명>
```

### 브랜치 삭제
```bash
# 로컬 브랜치 삭제
git branch -d <브랜치명>

# 강제 삭제 (병합 안 된 경우)
git branch -D <브랜치명>

# 원격 브랜치 삭제
git push origin --delete <브랜치명>
```

## 변경사항 확인

### Diff 확인
```bash
# 작업 디렉토리 vs 스테이징 영역
git diff

# 스테이징 영역 vs 마지막 커밋
git diff --staged

# 브랜치 간 비교
git diff <브랜치1> <브랜치2>

# 특정 파일만 비교
git diff <파일명>
```

### 로그 확인
```bash
# 커밋 히스토리
git log

# 한 줄로 보기
git log --oneline

# 그래프로 보기
git log --graph --oneline --all

# 최근 N개만
git log -n 5

# 특정 파일의 히스토리
git log <파일명>

# 통계 포함
git log --stat
```

## 원격 저장소 작업

### 원격 저장소 관리
```bash
# 원격 저장소 목록
git remote -v

# 원격 저장소 추가
git remote add origin <저장소-URL>

# 원격 저장소 제거
git remote remove <이름>

# 원격 저장소 이름 변경
git remote rename <기존이름> <새이름>
```

### Push (업로드)
```bash
# 원격에 푸시
git push origin <브랜치명>

# 최초 푸시 (upstream 설정)
git push -u origin <브랜치명>

# 이후 간단히
git push

# 강제 푸시 (주의!)
git push -f
```

### Pull (다운로드)
```bash
# 원격 변경사항 가져오기 + 병합
git pull

# 특정 브랜치에서
git pull origin <브랜치명>

# Rebase로 병합
git pull --rebase
```

### Fetch
```bash
# 원격 정보만 가져오기 (병합 안 함)
git fetch

# 모든 원격 브랜치
git fetch --all

# 가져온 후 수동 병합
git merge origin/main
```

## 병합 (Merge)

### 기본 병합
```bash
# 현재 브랜치에 다른 브랜치 병합
git merge <브랜치명>

# Fast-forward 방지 (병합 커밋 생성)
git merge --no-ff <브랜치명>

# 병합 취소
git merge --abort
```

### Rebase
```bash
# 현재 브랜치를 다른 브랜치 위로 재배치
git rebase <브랜치명>

# 대화형 rebase
git rebase -i HEAD~3

# Rebase 중단
git rebase --abort

# Rebase 계속
git rebase --continue
```

## 되돌리기

### 작업 디렉토리 변경사항 취소
```bash
# 특정 파일 되돌리기
git checkout -- <파일명>

# 모든 변경사항 되돌리기
git checkout -- .

# 새로운 방식
git restore <파일명>
```

### 스테이징 취소
```bash
# 특정 파일 unstage
git reset HEAD <파일명>

# 새로운 방식
git restore --staged <파일명>
```

### 커밋 되돌리기
```bash
# 마지막 커밋 취소 (변경사항 유지)
git reset --soft HEAD~1

# 마지막 커밋 취소 (스테이징도 취소)
git reset --mixed HEAD~1

# 마지막 커밋 완전히 삭제 (주의!)
git reset --hard HEAD~1

# 특정 커밋으로 이동
git reset --hard <커밋-해시>
```

### Revert (안전한 되돌리기)
```bash
# 특정 커밋을 되돌리는 새 커밋 생성
git revert <커밋-해시>

# 병합 커밋 되돌리기
git revert -m 1 <병합-커밋-해시>
```

## 임시 저장 (Stash)

### Stash 사용
```bash
# 현재 변경사항 임시 저장
git stash

# 메시지와 함께 저장
git stash save "작업 중인 기능"

# Stash 목록 확인
git stash list

# 가장 최근 stash 적용
git stash apply

# Stash 적용 + 삭제
git stash pop

# 특정 stash 적용
git stash apply stash@{1}

# Stash 삭제
git stash drop stash@{0}

# 모든 stash 삭제
git stash clear
```

## 태그 (Tag)

### 태그 생성/확인
```bash
# 태그 목록
git tag

# 태그 생성
git tag v1.0.0

# 주석이 있는 태그
git tag -a v1.0.0 -m "Version 1.0.0 release"

# 특정 커밋에 태그
git tag v1.0.0 <커밋-해시>

# 원격에 태그 푸시
git push origin v1.0.0

# 모든 태그 푸시
git push origin --tags

# 태그 삭제
git tag -d v1.0.0
git push origin --delete v1.0.0
```

## 유용한 명령어

### Clean
```bash
# 추적하지 않는 파일 확인
git clean -n

# 추적하지 않는 파일 삭제
git clean -f

# 디렉토리도 포함
git clean -fd
```

### Blame
```bash
# 각 줄의 마지막 수정자 확인
git blame <파일명>

# 특정 줄 범위만
git blame -L 10,20 <파일명>
```

### Show
```bash
# 특정 커밋 상세 정보
git show <커밋-해시>

# 특정 커밋의 특정 파일
git show <커밋-해시>:<파일명>
```

### Cherry-pick
```bash
# 특정 커밋만 가져오기
git cherry-pick <커밋-해시>

# 여러 커밋
git cherry-pick <커밋1> <커밋2>
```

## 단축 명령어 설정

```bash
# 자주 쓰는 명령어 단축
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'

# 사용 예시
git co main
git st
```

## 긴급 상황 대처

### 잘못 푸시한 경우
```bash
# 마지막 커밋 취소 (로컬)
git reset --hard HEAD~1

# 강제 푸시 (팀원과 조율 필수!)
git push -f origin <브랜치명>
```

### 실수로 삭제한 커밋 복구
```bash
# Reflog 확인
git reflog

# 특정 시점으로 복구
git reset --hard <커밋-해시>
```

### 잘못된 브랜치에서 작업한 경우
```bash
# 변경사항 stash
git stash

# 올바른 브랜치로 이동
git checkout <올바른-브랜치>

# Stash 적용
git stash pop
```

## 결론

이 명령어들을 익히면 Git을 효과적으로 사용할 수 있다. 처음에는 기본 명령어부터 시작하고, 점차 고급 기능을 익혀나가는 것이 좋다.

**꿀팁:**
- `git status`를 자주 확인할 것
- 커밋 전에 `git diff`로 변경사항 검토할 것
- 중요한 작업 전에는 브랜치를 만들 것
- 모르는 명령어는 `git <명령어> --help`로 확인할 것
