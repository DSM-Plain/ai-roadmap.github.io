## 개요

Repository(저장소)는 Git 프로젝트의 모든 파일과 버전 이력을 담는 저장 공간이다.

## 저장소의 종류

### 로컬 저장소 (Local Repository)
- 개발자의 컴퓨터에 있는 저장소
- 모든 Git 작업은 먼저 로컬에서 수행
- 네트워크 없이도 대부분의 작업 가능

### 원격 저장소 (Remote Repository)
- 서버에 있는 저장소
- Github, GitLab, Bitbucket 등
- 협업과 백업을 위해 사용

## 저장소 생성 방법

### 새로운 저장소 초기화
```bash
git init
```
- 현재 디렉토리를 Git 저장소로 만듦
- `.git` 폴더가 생성됨
- 빈 저장소에서 시작

### 기존 저장소 복제
```bash
git clone <저장소 URL>
```
- 원격 저장소를 로컬로 복사
- 모든 이력과 브랜치가 함께 복제됨
- 자동으로 원격 저장소와 연결

## 저장소 구조

### .git 디렉토리
```
.git/
├── HEAD          # 현재 체크아웃된 브랜치를 가리킴
├── config        # 저장소 설정
├── objects/      # 모든 커밋, 트리, 파일 내용
├── refs/         # 브랜치와 태그 참조
└── logs/         # 변경 이력
```

### 작업 디렉토리
- 실제 프로젝트 파일들
- `.git`을 제외한 모든 파일과 폴더
- 개발자가 직접 수정하는 공간

## 원격 저장소 관리

### 원격 저장소 추가
```bash
git remote add origin <URL>
```

### 원격 저장소 목록 확인
```bash
git remote -v
```

### 원격 저장소 이름 변경
```bash
git remote rename <기존이름> <새이름>
```

### 원격 저장소 제거
```bash
git remote remove <이름>
```

## Github 저장소 유형

### Public Repository (공개 저장소)
- 누구나 코드를 볼 수 있음
- 오픈소스 프로젝트에 적합
- 무료로 무제한 생성 가능
- 포트폴리오로 활용

### Private Repository (비공개 저장소)
- 초대받은 사람만 접근 가능
- 회사 프로젝트나 개인 작업용
- Github에서 무료로 제공
- 협업자 수 제한 없음

## 저장소 클론 vs 포크

### Clone (복제)
- 저장소를 그대로 복사
- 원본 저장소와 직접 연결됨
- 팀 프로젝트에서 주로 사용

### Fork (포크)
- 저장소를 내 계정으로 복사
- 독립적인 저장소 생성
- 오픈소스 기여 시 사용
- Pull Request로 원본에 기여

## 저장소 관리 팁

### README.md 작성
- 프로젝트 설명
- 설치 및 사용 방법
- 라이선스 정보
- 저장소의 첫인상

### .gitignore 설정
- 추적하지 않을 파일 지정
- 빌드 결과물, 로그 등 제외
- 보안 정보 보호

### LICENSE 파일
- 프로젝트 사용 조건
- MIT, Apache, GPL 등
- 오픈소스의 경우 필수

## 결론

Repository는 Git의 핵심 단위이다. 로컬과 원격 저장소의 개념을 이해하고, 적절히 관리하는 것이 협업의 시작이다.
